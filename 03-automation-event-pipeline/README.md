# 自动化事件流底座

## 产品定位

这个部分展示底层自动化能力：把外部输入转成标准事件，再进入任务队列、AI 判断、草稿输出和人工确认流程。

它不是主作品，但能解释为什么前两个 demo 可以继续扩展成更完整的运营系统。

## 设计原则

- 采集层只负责读取输入。
- 事件层只负责标准化和去重。
- 判断层只负责识别意图和生成草稿。
- 发送/执行层必须保留人工确认。
- 运行日志和真实消息不进入公开仓库。

## 安全流程

```text
外部输入
  -> 标准事件
  -> inbox
  -> AI / 规则判断
  -> draft / report
  -> 人工确认
  -> outbox / task
```

## 可复核点

- 能把复杂自动化拆成独立层。
- 知道哪些环节可以自动化，哪些必须人工确认。
- 能用事件流和队列降低系统耦合。
- 有安全边界意识，不把平台操作做成无审计全自动。
